Detailed Description

A frame stack stores all frames being executed by the JVM interpreter. The current context is referenced by the top of the stack.

Everytime a new method is called by the current context, a new frame is created and pushed into the top of the stack. When its execution finishes, the frame will be popped and its return value appended to the operand stack of the new top frame.

When the frame stack is empty, no context is currently being executed, indicating that the program has finished running.
